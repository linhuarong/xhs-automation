# 01_小红书发布工作流

## 目标

从 Feishu 发布池读取内容，通过 browser-worker 调度影刀 RPA，在跨境卫士账号/店铺环境中执行小红书发布 UI Flow，并形成可追踪 evidence。

## V2 基本链路

```text
Feishu 发布池
-> n8n 读取待发布记录
-> n8n 生成 publish_job
-> browser-worker 接收任务
-> Provider Router 选择 kuaijingvs_yingdao_rpa
-> 跨境卫士打开 account_id 对应账号/店铺环境
-> 影刀 RPA 执行发布 UI Flow
-> 影刀输出截图和 publish_evidence.json
-> browser-worker 读取 evidence
-> browser-worker 返回 WorkerResult
-> PostgreSQL / Feishu 承接发布状态
```

## browser-worker 职责

- 接收 `PublishJob`。
- 校验任务字段。
- 选择 provider。
- 调度跨境卫士环境启动。
- 调用影刀发布应用。
- 读取发布结果、截图和 evidence。
- 返回 WorkerResult。

browser-worker 不再作为主链路直接进入发布页、上传、填写和点击发布。`selenium_chrome` 仅用于本地 debug。

## 影刀 RPA 职责

- 在跨境卫士环境中执行小红书发布 UI Flow。
- 上传图片。
- 填写标题、正文、标签。
- 执行提交前检查。
- 需要人工确认时暂停并输出 `waiting_human_verification`。
- 成功或失败都输出 `publish_evidence.json`。

## 执行要求

- 只能使用真实浏览器页面 UI Flow。
- 不允许调用小红书未授权接口。
- 每次任务必须保留发布前、发布中、发布后关键截图。
- 失败必须保留异常截图和错误日志。
- 支持人工审核状态控制。
- 遇到登录、验证码、扫码、安全确认、风控或账号限制时返回 `waiting_human_verification`。

## 禁止事项

- 不逆向 request。
- 不伪造 XHR、fetch、token、cookie、sign。
- 不绕过登录、验证码、扫码、安全确认、风控或账号限制。
- 不实现验证码自动处理。
- 不隐藏自动化特征或规避平台检测。
