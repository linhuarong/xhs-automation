# 02_小红书热点追踪工作流

## 目标

基于 Feishu 关键词池，定时在小红书页面搜索关键词，截取搜索结果页，提取前 N 条结果，并形成热点追踪记录。

## 基本链路

Feishu 关键词池
→ n8n 定时任务
→ 生成 search_job
→ browser-worker 打开小红书搜索页
→ 输入关键词
→ 等待结果加载
→ 截取搜索结果页
→ 提取前 N 条结果
→ 必要时进入详情页截图
→ 上传截图到 MinIO
→ 写入 PostgreSQL
→ 回写 Feishu 热点池

## 需要记录的字段

- job_id
- keyword
- search_time
- rank
- title
- author
- note_url 或页面标识
- like_count
- comment_count
- collect_count
- screenshot_path
- detail_screenshot_path
- raw_text
- status
- error_message

## 排名判断原则

- 以浏览器实际页面展示顺序为准。
- 每条结果记录 rank。
- 如果页面出现广告、推荐、置顶等特殊标记，需要记录 result_type。
- 如果页面加载异常，必须截图留证。

## 截图策略

第一阶段：
- 先截搜索结果页。
- 前 N 条结果做结构化提取。
- 对重点结果再进入详情页截图。

第二阶段：
- 增加标题、正文、标签、评论、互动数据提取。
- 增加多关键词趋势对比。
