# 07_n8n工作流规范

## 定位

n8n 负责任务编排，不直接执行复杂浏览器操作。

## 主要职责

- 定时读取 Feishu 关键词池
- 生成人工或自动触发任务
- 调用 browser-worker
- 接收执行结果
- 回写 Feishu
- 记录异常分支
- 在 V2 主链路中，n8n 不直接调用影刀或跨境卫士；统一通过 browser-worker 调度。

## 规则

- n8n 不保存大文件。
- 图片和截图统一进入 MinIO。
- n8n 节点必须保留错误处理分支。
- 每个 workflow 必须有输入输出说明。
- n8n 应消费 WorkerResult 中的 `evidence_json_path` 和 `normalized_records`，再回写 Feishu。
